# 02｜建立最终快照

## 1. 目标

在任何清理动作前，固定 Vault OS 的可回溯状态，记录当前代码、运行方式、最终资产和敏感信息检查结果。

## 2. 前置条件

- 01-scope-and-freeze 已完成；
- 用户已确认关闭范围；
- 用户已确认清理前标签名称或授权本阶段提出并建立该标签。

## 3. 允许动作

- 读取仓库和运行配置；
- 创建 docs/archive/01-final-snapshot.md；
- 在用户已授权的前提下建立清理前 Git 标签；
- 执行非破坏性的构建、测试、启动或静态检查；
- 记录资产位置、文件哈希和验证结果；
- 更新 PROJECT_CLOSURE_STATE.md。

## 4. 禁止动作

- 删除、移动或修改业务文件；
- 修复缺陷、升级依赖或重构代码；
- 上传本地资产、创建公开 Release 或修改仓库可见性；
- 输出、复制或记录真实密钥、个人数据与真实对话内容。

## 5. 执行步骤

1. 记录仓库 URL（如有）、当前分支、HEAD 提交号、Git 状态、最近版本和标签。
2. 记录项目的环境要求、依赖管理工具、安装、Build、Test 和最小启动命令。
3. 若已有脚本，执行最小的非破坏性验证，并完整记录命令、时间、结果和限制。
4. 识别最终资产：
   - 可运行版本或可复现构建入口；
   - 当前实际 UI 的截图/录屏；
   - Figma 文件及四张关键帧的链接或导出位置；
   - Eval、测试、Trace、夹具等真实证据；
   - 安装包、Release、版本号和 SHA-256（如存在）。
5. 检查 .env、配置、日志、截图、样例数据、数据库导出和 Git 历史中可能的敏感信息线索。只记录位置和处置建议。
6. 建立或记录清理前标签。
7. 填写 01-final-snapshot.md，并更新状态文件。

### 6.4 敏感信息检查

只记录以下内容：位置、风险类别、建议处置和是否已处理。不得写入密钥、Token、个人数据、真实聊天内容或数据库正文。

### 6.5 快照结论

说明当前版本是否已实际验证可运行；若不能运行，必须写明障碍、影响和可复现条件。

## 7. 退出条件
