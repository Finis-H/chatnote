# 06｜外部资源、安全与运行环境收尾

## 1. 目标

识别并安全处理 Vault OS 在仓库之外仍可能存在的资源、密钥、费用、自动任务、数据与对外入口，避免项目归档后仍有安全或成本风险。

## 2. 核心原则

本工作包分为“检查”与“执行”两部分：

- 检查可在不改变外部状态的前提下进行；
- 停止服务、删除数据、撤销密钥、修改仓库可见性、取消订阅或产生费用影响的动作，必须逐项获得用户确认。

## 3. 允许动作

- 审查代码、配置、CI、文档和账号清单中对外部资源的引用；
- 创建 docs/archive/07-decommission-checklist.md；
- 记录资源名称、用途、风险、推荐动作和用户确认状态；
- 在用户已经明确批准具体动作后，执行该动作并记录结果；
- 更新 PROJECT_CLOSURE_STATE.md。

## 4. 禁止动作

- 未经确认关闭、删除或变更任何外部服务；
- 输出 API Key、Token、连接字符串、个人数据或账户敏感信息；
- 删除本地数据库、备份、聊天记录或用户数据；
- 假设“未在代码中发现”就等于“外部资源不存在”。

## 5. 检查清单

### 5.1 密钥与访问

- .env、示例配置、CI Secrets、部署变量、文档、Git 历史；
- LLM/API Provider、OAuth、Webhook、插件凭据、数据库/存储凭据；
- 本地桌面应用或浏览器中的持久化 Token；
- 推荐动作：撤销、轮换、保留、确认不存在。

### 5.2 部署与费用

- 静态站点、Server、Container、Cloud Function、数据库、对象存储；
- 域名、DNS、SSL 证书、监控、日志、错误追踪；
- CI/CD、定时任务、Cron、自动发布、Preview 环境；
- 付费订阅、试用额度与账单告警。

### 5.3 数据与隐私

- 本地长期记忆、聊天记录、Trace、日志、测试夹具、截图、录屏；
- 数据库与本地文件目录；
- 是否包含真实用户、第三方内容、隐私或受版权保护资料；
- 保留期限、加密/访问控制、删除/匿名化决定。

### 5.4 对外状态

- GitHub 仓库可见性、Issues、Discussions、Release、Pages；
- README 是否声明归档状态；
- Figma 文件和原型分享权限；
- 作品集演示入口及其数据脱敏状态。

## 6. 归档产物模板：07-decommission-checklist.md

| ID | 资源类别 | 资源名称/位置 | 用途 | 当前状态 | 推荐动作 | 风险 | 用户确认 | 执行结果 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| EXT-001 | API Key | 仅记录位置，不记录值 | 待填写 | 待确认 | 撤销/轮换/保留 | 高 | 待确认 | 未执行 |

“推荐动作”只能使用：

- 检查后保留；
- 撤销或轮换；
- 停止运行；
- 关闭并保留备份；
- 删除；
- 无法确认，需用户检查。

## 7. 执行记录

对每一个已批准外部动作，记录：

- 用户批准的范围与时间；
- 操作时间；
- 执行结果；
- 是否存在不可逆影响；
- 费用、访问、数据保留和恢复方式；
- 后续验证。

## 8. 退出条件

每个资源均处于“已处理”“明确保留”或“需要用户线下确认”的状态；不存在未记录的自动运行、付费或有效密钥风险。
