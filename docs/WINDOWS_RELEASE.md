## 预期使用方式（未验证）

如后续验证通过，目标 Windows 11 x64 使用方式为运行 `Vault OS_0.1.0_x64-setup.exe`，按安装向导完成安装后打开 Vault OS。候选构建路径旨在将桌面界面和本地后端一并打包；在实际验证前，不能据此断言无需 Python、Node.js、npm、虚拟环境或额外服务。

候选实现预期在首次启动时进入“引擎设置”。默认配置面向千问兼容接口；若使用其他 OpenAI 兼容服务，预期可在同一页填写服务提供的 Base URL、主模型和轻量模型。该启动与配置流程尚未在本阶段实际验证。

若后续验证候选安装包，首次模型调用预计需要联网并使用用户自己的模型服务。Windows 11 的 WebView2 运行组件兼容性亦待实际安装验证。

## 预期本地数据与卸载边界（未验证）

候选源码将应用数据路径指向当前 Windows 用户目录：`%LOCALAPPDATA%\Vault OS\vault`；其预期内容包括 API 配置、聊天记录、记忆、知识库、Trace 和插件数据。该路径、安装目录写入边界及网络行为尚未经过安装包验证。

候选源码意图使卸载不主动删除该目录，以避免误删用户数据；实际卸载行为尚未验证。后续如经人工确认需要彻底清理，应先备份需要保留的内容，再手动删除这一明确目录。

```powershell
.\scripts\build-windows-release.ps1
```
