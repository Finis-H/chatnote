# Vault OS 产品说明

> 文档定位：本文是 Vault OS 的产品定义文档，说明产品问题、原则、当前能力、边界和 Roadmap。项目入口页见 [../README.md](../README.md)，开发者安装与运行见 [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md)，作品集案例见 [PRODUCT_CASE.md](PRODUCT_CASE.md)。

## 一句话定位

Vault OS 是一个本地 AI 工作台，用于长期记忆、知识检索、插件执行和可观测 AI 协作。

它不是普通聊天机器人，也不是开箱即用的商业 SaaS。当前形态更接近一个可持续演进的本地桌面 AI 系统：让用户在本地掌控资料、记忆、工具、插件和执行过程。

## 产品问题

普通 AI 聊天工具在长期使用中容易遇到几个断点：

- 上下文容易丢失，用户需要反复交代背景。
- 工具调用不透明，用户很难判断 AI 正在做什么、是否失败、哪里需要介入。
- 记忆不可控，临时请求、偏好、事实和敏感信息容易混在一起。
- 插件和外部工具的权限边界不清晰，扩展能力与安全边界容易互相拉扯。
- 桌面工作流集成弱，知识、配置、插件、执行过程分散在不同入口。

Vault OS 尝试把这些能力收束为一个本地工作台：对话可以长期协作，工具执行可以观察，记忆写入需要边界，插件扩展必须有权限约束。

## 产品原则

- **Local-first / 用户掌控数据**：运行时数据围绕本地 `vault/` 组织，配置、聊天历史、Trace、知识库、向量库和插件数据都应便于掌控、迁移和审计。
- **Long-term memory / 长期协作**：系统维护画像、实体关系、偏好、事件源和认知快照，但必须区分长期事实与一次性请求。
- **Observable execution / 执行可观测**：复杂任务不能只给最终回复，应该展示规划、工具调用、状态变化、异常和结果。
- **Permission-aware / 权限边界清晰**：插件能力需要声明权限，敏感权限需要确认，第三方插件输出不能被当作可信系统指令。
- **Extensible tools / 插件可扩展**：通过 VPM 插件、工具注册和动态 Vue 面板，把领域能力接入同一个桌面工作台。
- **Calm cyberpunk / 克制、专业、轻赛博**：保留暗色终端感和轻赛博识别度，但在记忆、权限、删除、配置重载等高信任流程中使用克制、明确、可审计的表达。

## 当前核心能力

### 主控终端

主控终端是日常对话和任务入口，承载主会话与临时会话。主会话使用长期记忆和历史上下文；临时会话用于隔离探索，不读取主记忆、不写持久状态，结束后会丢弃迟到结果。

当前前端已将左侧导航分为执行、知识、系统等工作区，并突出主控终端、临时会话、记忆同步、画像导入、设置和插件中心。

### Command Console

底部输入区已经从普通输入框收敛为 Command Console 形态：保留命令提示符、发送按钮、回车提示和焦点状态。它仍保持轻量，不在所有视图中扩展复杂上下文提示；当前只在笔记详情等强相关场景显示上下文提示。

### Trace 观测

后端记录 trace/span、状态、耗时和事件，前端主控终端内置可折叠 Trace 面板。当前 Trace 面板能够展示执行步骤、状态、层级、耗时和失败/降级等状态，是可观测执行的基础版本。

更完整的阶段摘要、错误解释、恢复建议和归档视图仍属于后续方向。

### AgentDock / 插件任务面板

AgentDock 是当前插件/Agent 面板宿主，用于承载后端通过 `ui_command` 或插件消息打开的动态 Vue 面板。它支持 dock、mini、focus 等形态，包含插件来源、任务状态、授权待处理状态、关闭和聚焦等控制。

这让插件 UI 不再只是临时侧栏，而是主控体验的一部分。当前实现已经可用，但插件任务状态、权限语义和更多面板类型仍有继续打磨空间。

### 知识 / RAG

系统使用 ChromaDB 持久化向量库，并通过 RAG 组装器把检索上下文与画像、关系、认知快照等信息合并进系统提示词。前端提供新闻信息、订阅收藏、笔记详情等知识入口；插件也可以提供知识材料。

### 记忆相关能力

记忆系统基于 SQLite/SQLModel，包含实体、关系、事件源、待审项、L2 实体快照和认知快照。`memory_rules.py` 会过滤搜索、播放、删除、推荐等瞬时交互，避免误写长期记忆。

画像导入采用两阶段流程：先标准化并预览，再提交到记忆流程。当前 UI 已把“写入记忆”类文案收敛为“提交到记忆待审”等更准确表达。

### 设置与插件中心

设置页负责模型、base URL、API Key、embedding 等配置管理，当前高风险配置按钮已收敛为“保存并重载配置”。插件中心展示 VPM 插件，支持管理入口和卸载确认。

插件系统支持 manifest、工具注册、动态 Vue 面板、敏感权限确认和第三方输出隔离。第三方插件不能仅靠 manifest 自称 first-party 获得信任。

### Tauri 桌面形态

前端基于 Vue 3 + Vite + Tauri v2。桌面壳提供窗口、快捷键、生产模式 sidecar 启动和本地运行体验。后端启动后会写入 `.server_port` 和 `.run_token`，前端通过 Tauri command 读取后连接 WebSocket。

## 本次产品化改版重点

本次产品化改版的目标，是把 Vault OS 从“赛博主题 Demo”收敛为更专业、可信、可维护的个人 AI 工作台。改版重点集中在前端表达层，不改变 WebSocket、记忆流程、插件权限流程、RAG 摄入、Trace 数据结构或工具执行逻辑。

已落地或阶段性落地的重点包括：

- **导航分组**：左侧导航从扁平功能列表收敛为执行、知识、系统等分组，使用更统一的图标和状态表达。
- **设计 token**：`chat-ui/src/assets/cyber-theme.css` 已建立颜色、字体、背景层级、边框、圆角、间距、阴影、focus 和动效 token。
- **Command Console**：底部输入区完成主控命令入口的视觉收敛，保留轻量、直接的工作台气质。
- **Trace 基础观测面板**：主控终端中的 Trace 面板可展示执行步骤、状态、层级和耗时。
- **AgentDock 产品化**：插件面板宿主已具备来源、状态、mini/dock/focus 模式和任务面板结构。
- **可见 focus ring**：全局恢复并统一 `focus-visible`，避免桌面工具丢失键盘可用性。
- **高风险/权限/记忆文案收敛**：删除、插件卸载、配置重载、画像导入、记忆待审、临时会话结束等流程改为更克制、明确、可审计的表达。

## 当前边界

当前不能写成“全部完成”。准确状态如下：

- **主控体验 v1 的产品化骨架已阶段性完成**：主控终端、分组导航、Command Console、Trace 基础观测面板和 AgentDock 已形成稳定骨架。
- **全站业务页一致性仍在推进**：知识列表和记忆同步页已接入部分通用组件，但设置页、画像导入页、插件中心等仍有继续统一空间。
- **高风险确认弹窗已有限落地，仍需体系化扩展**：档案删除和插件卸载已接入 `DangerDialog`；插件权限确认目前主要是文案收敛，尚未完全统一为同一弹窗体系。
- **基础组件抽象已试点，仍属于下一阶段重点**：`PageFrame`、`SegmentedControl`、`VaultCard` 已用于部分页面，但还不是完整 design system，也没有覆盖所有业务页和状态。
- **Trace 仍是基础观测面板**：可以显示执行步骤和状态，但阶段摘要、错误解释、恢复建议和可归档复盘仍未完成。
- **作品集材料尚未完整归档**：已有实施报告和截图占位，但 before/after 截图、案例页面和完整展示叙事仍需补齐。

## 开发路线 Roadmap

### Now

- 稳定主控体验 v1 的产品化骨架：主控终端、Command Console、Trace 基础观测面板和 AgentDock。
- 同步 README 与 PRODUCT 的公开说明口径，避免文档夸大或落后于代码。
- 保持当前改版边界：优先表达层和轻量组件复用，不混入业务协议变更。

### Next

- 统一危险确认：继续完善 `DangerDialog` 在删除、卸载、权限、配置重载等高风险场景中的适用边界。
- 推进基础组件抽象：扩大 `PageFrame`、`SegmentedControl`、`VaultCard` 的覆盖范围，减少重复 frame/filter/card 样式。
- 提升业务页一致性：设置、画像导入、插件中心、记忆卡片等页面继续向同一设计 token 和组件体系收敛。
- 优化 Trace 可读性：增加阶段摘要、失败解释和下一步建议。

### Later

- 更完整的 Trace 归档和任务复盘能力。
- 插件市场或插件中心的可信体系、备份/导出和更清晰的权限展示。
- 更成熟的记忆审核体验，包括来源、原因、影响范围和确认历史。
- 知识管理体验继续打磨，包括资料摄入、检索反馈和插件知识材料管理。
- 作品集展示页或案例材料，补齐 before/after 截图、设计取舍和阶段成果。
