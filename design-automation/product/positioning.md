# Product Positioning

## One-line definition

Vault OS is a local-first AI workstation for sustained collaboration: it combines long-term memory, local knowledge retrieval (RAG), controlled tool and plugin execution, and observable task traces in a desktop workspace.

## Problem

Conventional chat surfaces lose context, conceal tool activity, blur transient requests with durable memory, and make plugin permissions difficult to judge. Vault OS turns those fragmented concerns into a workspace where users can understand what the system is doing and retain control of their local data.

## Portfolio message

The Figma portfolio should present Vault OS as a calm, professional, lightly cyber-styled control surface—not a generic chatbot or a neon demo. The primary user is an AI-heavy knowledge worker or developer who needs repeatable local workflows.

## Product principles

- **Local-first:** configuration, chat history, traces, knowledge, vector data, and plugin data are organized under the local `vault/` root.
- **Deliberate memory:** durable facts are separated from one-off prompts and flow through a review-oriented memory process.
- **Observable execution:** complex work exposes task stages, tool activity, state, and outcomes instead of only a final answer.
- **Permission-aware extension:** plugin capabilities are declared and sensitive third-party actions require user involvement.
- **Desktop control surface:** the Command Console, Trace, and AgentDock are the primary operating experience.

## Non-goals for this directory

This portfolio brief does not claim production metrics, completed evaluation results, or behavior beyond the current repository. It also does not authorize UI implementation.
