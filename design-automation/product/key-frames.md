# Four Portfolio Key Frames

All frames use a fixed **1440 × 900 px** canvas. The portfolio has exactly these four frames—no additional key frames are in scope.

| # | Frame | Story to communicate | Required states and evidence |
| --- | --- | --- | --- |
| 1 | Command Console | Vault OS is a local AI workstation, not a chat-only surface. | Main-session task entry, connection state, visible command focus, grouped workspace navigation, and a concise local-first framing. |
| 2 | Agent execution and Trace | Tool-using work is observable while it runs. | A running task, readable Trace stages, status progression, timing or outcome information, and a clear path back to the command surface. |
| 3 | Memory staging | Durable memory is reviewed rather than silently inferred. | A low-confidence or conflict candidate, source/impact context where available, explicit accept/reject actions, and the three-day countdown. |
| 4 | Third-party plugin permission | Extension does not bypass consent or trust boundaries. | Plugin identity, requested sensitive permissions, purpose/risk explanation, redacted argument preview, and deny / allow-once / allow-for-session choices. |

## Cross-frame rules

- Preserve the dark, restrained, professional visual language; avoid broad high-intensity neon.
- Keep keyboard focus visibly represented on actionable controls.
- State labels must distinguish equivalent `idle`, `running`, `success`, `failed`, and `needs_permission` conditions.
- Frame 3 and Frame 4 are high-trust workflows: use precise, auditable language rather than playful copy.
